Import Students

CSV File Format

Your CSV file should have the following columns (in this order):

  • student_code - Student registration number (optional). If not provided, system auto-generates next number. Must be unique if provided.
  • full_name - Student's full name (required)
  • phone - Student's phone number (required)
  • gender - Gender (optional). Allowed values: Male, Female, Other
  • email - Student's email (optional)
  • address - Student's address (optional)
  • education_level - Education level (optional). Allowed values: School, Pre-University, Diploma, Technical, Undergraduate, Postgraduate
  • qualification - Qualification (optional)
  • employment_status - Employment status (optional). Allowed values: unemployed, employed, self_employed, student
  • status - Student status (optional, defaults to active). Allowed values: active, completed, dropped
  • branch_code - Branch code (optional, defaults to Head Office if not provided). Use the branch codes from the list below.

Note: The first row will be treated as headers. Duplicate registration numbers are automatically skipped to prevent duplicate student records. If no registration number is provided, the system auto-generates the next number.
